Uses of final keyword in Java
1) To Create Constant Field
•	If a field is declared as final, its value cannot be changed once initialized.
•	It must be initialized at the time of declaration or in the constructor.
Example:
class MathConstants {
    final static double PI = 3.14;  // constant field
    public static void main(String[] args) {
        System.out.println("Value of PI: " + PI);
        // PI = 3.1415; // Error: cannot assign a value to final variable PI
    }
}
________________________________________
2) To Prevent Method Overriding
•	Sometimes, we want to prevent a method from being overridden in subclasses.
•	Use final keyword before the method declaration in the superclass.
Example:
class A {
    final void display() {
        System.out.println("This is a final method");
    }
}

class B extends A {
    // void display() {   // Error: cannot override final method
    //     System.out.println("Illegal attempt");
    // }
}

class TestFinalMethod {
    public static void main(String[] args) {
        B b = new B();
        b.display(); // calls final method from A
    }
}
________________________________________
3) To Prevent Inheritance
•	Declaring a class as final prevents it from being subclassed.
•	All methods in a final class are implicitly final.
Example:
final class A {
    void show() {
        System.out.println("This is a final class");
    }
}

// class B extends A {  // Error: cannot inherit from final class
// }

class TestFinalClass {
    public static void main(String[] args) {
        A a = new A();
        a.show();
    }
}
